# 2. Problem and goals

**Canonical:** [prd.md](../../_bmad-output/planning-artifacts/prd.md) · [product-brief.md](../../_bmad-output/planning-artifacts/product-brief.md)

## Problem

- Users want to **buy and sell items in crypto** without an unnecessary fiat conversion step (tax and friction, e.g. Ukraine).
- Informal P2P deals lack structured recourse.
- Crypto products optimize for asset trading, not classifieds.

## Goals

| Phase | Goal |
|-------|------|
| 1 | OLX core + coordinated stablecoin trades |
| 2 | Payment verification + one fulfillment path proven |
| 3 | On-chain escrow; optional HTLC if approved |

## Non-goals (MVP)

- Custodial fiat, email/password auth, marketing “trustless escrow” while funds are off-chain.
